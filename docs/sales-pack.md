# Sales-pack: продажи и маркетинг

Технический и управленческий контур в репозитории уже есть: [content-pack.md](./content-pack.md) → [final-release-checklist.md](./final-release-checklist.md) → [план-после-запуска.md](./план-после-запуска.md).

**Узкое место сейчас не разработка**, а материалы от владельца бизнеса. Пока они в пути, продажи и охват можно готовить отдельно — этим и служит этот документ.

---

## Два сценария

### Сценарий A — ждём материалы

Когда данные пришли:

1. Закрыть [content-pack.md](./content-pack.md) по чеклисту  
2. Подставить контент на сайт  
3. QA  
4. Deploy  
5. Publish (индексация по [final-release-checklist.md](./final-release-checklist.md))

### Сценарий B — не ждать, готовить продажи

Параллельно со сбором контента-пакета можно вести этот файл:

- тексты для Instagram  
- офферы под рекламу  
- шаблоны ответов в WhatsApp  
- описание для Google Maps  
- promo-посты  
- акции и спецпредложения

## Ответы клиентам (WhatsApp и др.)

Шаблоны, чтобы отвечать быстро и единообразно. Подставьте цены и сроки из контент-пакета, когда они будут.

**Первый контакт / приветствие**

```
Здравствуйте! Спасибо за интерес к [название]. 
Уточните, пожалуйста: на какую дату нужен торт и для скольки персон?
```

**Запрос цены**

```
По стоимости: от ___ ₽ за [единица]. Точнее смогу сказать после [уточнение: вес / дизайн].
```

**Сроки**

```
Стандартный срок заказа — ___ дней. Раньше — по согласованию, возможна доплата.
```

**Доставка**

```
Доставка: [зоны / стоимость]. Самовывоз: [адрес / время].
```

**Дополнительные блоки**

```
(закрытие возражения / бронь / предоплата / напоминание о макете)
```
